# 인코딩 문서

이 패키지는 영수증 텍스트용 공개 타입으로 고정된 `ReceiptEncoding` 값을 제공합니다

런타임 변환은 `utf8`은 `TextEncoder`, `ascii`는 ASCII encoder, 나머지 값은 `iconv-lite`를 사용합니다

## 인코딩과 code page

`encoding(value)`는 JavaScript 문자열을 어떤 바이트로 바꿀지 정합니다

`codePage(page)`는 ESC/POS `ESC t n` 명령을 보내 프린터의 문자 테이블을 선택합니다

프린터 매뉴얼에서 같은 문자 테이블 선택이 필요하다고 안내할 때만 둘을 함께 사용합니다

```ts
const receipt = createReceipt()
  .initialize()
  .codePage(21)
  .encoding("cp949")
  .text("테스트 출력")
  .encode();
```

ESC/POS code page 번호는 프린터 모델, 펌웨어, 판매 지역, emulation mode에 따라 달라질 수 있습니다

아래 값은 일반적인 후보로만 보고, 실제 프린터 매뉴얼이나 테스트 출력으로 확인해야 합니다

## 지원 인코딩

| 인코딩         | 주 사용 국가와 언어                 | 일반적인 ESC/POS code page 후보              | 참고                                      |
| -------------- | ----------------------------------- | -------------------------------------------- | ----------------------------------------- |
| `utf8`         | 프린터가 UTF-8을 직접 지원하는 경우 | 없음                                         | 많은 ESC/POS 프린터는 UTF-8 바이트를 직접 표시하지 못함 |
| `ascii`        | 영어, 숫자, 단순 기호               | 없음                                         | 대부분의 문자 테이블에서 안전한 부분집합  |
| `cp437`        | 미국, 표준 서유럽                   | `0`                                          | PC437로 표기되는 경우가 많음              |
| `cp850`        | 서유럽                              | `2`                                          | PC850 multilingual로 표기되는 경우가 많음 |
| `cp852`        | 중부와 동유럽 라틴 문자             | `18`, 일부 기기는 `6`                         | 폴란드어, 체코어, 슬로바키아어, 헝가리어 |
| `cp858`        | 유로 기호가 필요한 서유럽           | `19`                                         | PC850에 유로 기호를 더한 계열             |
| `cp860`        | 포르투갈어                          | `3`                                          | DOS Portuguese                            |
| `cp863`        | 캐나다 프랑스어                     | `4`                                          | DOS Canadian French                       |
| `cp865`        | 북유럽 언어                         | `5`                                          | 덴마크어, 노르웨이어 등                   |
| `cp866`        | 키릴 문자                           | `17`, 일부 기기는 `7`                         | 러시아어와 기타 키릴 문자                 |
| `cp949`        | 한국어                              | 모델별 지정, 한국 판매 프린터 매뉴얼 확인 필요 | 프린터 매뉴얼의 값을 우선 사용            |
| `cp932`        | 일본어                              | 모델별 지정, 일부 profile에서는 `1`           | `ESC t`만으로 부족하고 일본어 모드가 필요할 수 있음 |
| `cp950`        | 번체 중국어                         | 모델별 지정                                  | 대만, 홍콩에서 흔히 사용                  |
| `big5`         | 번체 중국어                         | 모델별 지정                                  | `cp950`과 사용 지역이 비슷함              |
| `gb18030`      | 간체 중국어                         | 모델별 지정                                  | 중국 본토에서 흔히 사용                   |
| `windows-874`  | 태국어                              | 모델별 지정, Thai table 변형 확인 필요       | 정확한 Thai table 번호 확인 필요          |
| `windows-1250` | 중부와 동유럽 라틴 문자             | 모델별 지정                                  | Windows ANSI code page                    |
| `windows-1251` | 키릴 문자                           | 모델별 지정                                  | Windows ANSI code page                    |
| `windows-1252` | 서유럽과 미주 라틴 문자             | `16`                                         | WPC1252로 표기되는 경우가 많음            |
| `windows-1253` | 그리스어                            | 모델별 지정                                  | Windows ANSI code page                    |
| `windows-1254` | 터키어                              | 모델별 지정                                  | Windows ANSI code page                    |
| `windows-1255` | 히브리어                            | 모델별 지정                                  | Windows ANSI code page                    |
| `windows-1256` | 아랍어                              | 모델별 지정                                  | Windows ANSI code page                    |
| `windows-1257` | 발트어                              | 모델별 지정, 일부 기기는 `25`                 | 리투아니아어, 라트비아어, 에스토니아어    |
| `windows-1258` | 베트남어                            | 모델별 지정                                  | Windows ANSI code page                    |

## 선택 기준

한글 영수증은 `encoding: "cp949"`로 시작하고, code page 값은 프린터 제조사 문서에 적힌 값을 사용합니다

영문만 출력하면 `ascii` 또는 기본값 `utf8`을 사용할 수 있습니다

서유럽 언어는 프린터 문자 테이블에 따라 `cp850`, `cp858`, `windows-1252`를 먼저 확인합니다

CJK 프린터는 일반 `ESC t n` 테이블인지, 별도 언어 모드가 필요한지, 드라이버 변환을 쓰는지 확인합니다

글자가 깨질 때는 다음 순서로 확인합니다

1. `encoding(value)`가 보내는 바이트 확인
2. `codePage(page)`로 선택한 프린터 문자 테이블 확인
3. 프린터가 드라이버 변환 없이 RAW 바이트를 받는지 확인
4. 프린터 펌웨어가 해당 언어 테이블을 지원하는지 확인
